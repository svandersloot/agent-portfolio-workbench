Plain-English Demonstration

Runtime Context Map

A small routing guide that helps a live ROVO agent use the right source for the right task, instead of trying to read every governance page every time.

1. Agent Lab

The builder workshop. This is where owners iterate, compare versions, score outputs, run regression prompts, and prepare promotion materials. It can be richer and messier because normal users do not live here.

2. Runtime Context Map

The lightweight routing map. It says, "For this kind of user request, use this source page first, avoid these substitutes, and ask for clarification if the source is missing."

3. Studio Promotion Packet

The human setup handoff. It includes copy-ready Studio instructions, required knowledge sources, test prompts, smoke checks, and a checklist for the person configuring ROVO Studio.

How It Works

Builders maintain rich source material behind the scenes. The live agent gets a narrow map, so it knows which approved page to use for each task.

1

User asks the live ROVO agent for help.

2

The agent checks the runtime context map.

3

The map points to the smallest trusted source.

4

The agent answers with focused context or asks for missing evidence.

What A Map Might Look Like

User Task Use This Source First Do Not Substitute If Missing
Draft deployment notes Deployment Notes Standard Generic Jira quality guidance Ask for release scope and source link
Review ticket quality Team Jira Standards Release notes templates Ask which team standard applies
Assess release readiness Release assessment template and Jira evidence Unlinked assumptions Mark evidence missing instead of guessing

The goal is not to make live agents read everything. The goal is to let builders maintain a rich workshop while live agents retrieve only the specific context needed.

Demonstration only. This preview explains a proposed next slice; it does not claim Runtime Context Maps are already implemented for every agent.
