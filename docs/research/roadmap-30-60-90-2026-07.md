# 30/60/90 Roadmap, Top 10 Build Items, Top 10 Risks

Date: 2026-07-08 · Companion to `enterprise-agent-governance-operating-model-2026-07.md` · All items are recommendations; promotion, ownership, and visibility decisions remain human-owned.

## Days 1-30 — Prove the loop on one team

| Do | Detail |
|---|---|
| Run the Stage 5 pilot readiness review and start the first pilot | 1-2 agents, named users, feedback channel. Everything else is secondary to one real pilot with real numbers. |
| Collapse process to five gates with risk tiers | One-page intake; Tier 1 self-service checklist. Update Stage 5 docs to point at the gates rather than duplicating them. |
| Make the eval pack executable | Load the regression prompt packs into Rovo Agent Evaluations (CSV ≤50 prompts) so evaluation is a run, not a document. Record results as evidence. |
| Baseline measurement | Turn on/record Rovo per-agent usage analytics; sample accepted/edited/rejected weekly. |
| Stop | Backfilling page families for non-pilot agents; new templates; any new governance doc that isn't a pointer. |

Evidence to collect: intake-to-pilot elapsed time; eval pass rates; weekly acceptance sampling; at least one reuse decision that prevented a duplicate build.

## Days 31-60 — Make it an org capability, not a personal practice

| Do | Detail |
|---|---|
| Move to org-hosted Git with a second maintainer | The single highest de-risking action. Checks run in CI (pwsh on runners), not on one laptop. |
| Extract the platform-neutral registry record | Core ~16 fields + platform extension blocks; regenerate the completeness report from it. |
| Drift check as scheduled routine | Re-capture pilot agents on cadence; diff; log reconciliations. Also rerun eval packs after any Atlassian model-update announcement. |
| Publish the leadership snapshot from pilot data | First monthly one-pager with real numbers. Present it. |
| Keep manual | Studio configuration; all approvals; pilot user selection. |

Evidence: a teammate completes one full intake→pilot cycle without your live help; snapshot delivered to a manager; first drift caught and reconciled.

## Days 61-90 — Prove cross-platform and pitch

| Do | Detail |
|---|---|
| Register one non-Rovo agent | A Copilot Studio agent (use native solutions/Git ALM as its config path) or a Claude skill/managed-MCP config. One real row proves the registry generalizes. |
| Demo for leadership | 15 minutes: pain point → intake → reuse check → pilot metrics → snapshot. Lead with the time given back, close with the audit trail. |
| Propose the risk-tier policy for adoption | The one governance ask that scales you out of being the bottleneck. |
| Watch for platform events | If Rovo versioning/drafts or sprawl management ships, adopt it and retire the capture workaround for whatever it covers. |

Evidence: second-platform registry row; demo delivered; leadership decision on hosting/tiering asks.

## Top 10 next build items (ranked by ROI)

1. First team pilot executed with the feedback tracker actually filled (everything depends on this).
2. Rovo Evaluations CSV export of each regression prompt pack + recorded results.
3. Five-gate/risk-tier consolidation of the lifecycle docs (delete-or-pointer pass included).
4. Platform-neutral agent registry file + schema v2 with platform extension blocks.
5. CI job for existing checks (private-data scan, page-family consistency, stale handoffs, schema validation).
6. Scheduled drift-diff routine over normalized captures, with a one-line drift log.
7. Monthly leadership snapshot generated from registry + analytics + tracker inputs.
8. One-page Tier 1 self-service checklist (builder can ship a personal read-only helper without you).
9. Second-platform registry row + capture/normalize adapter (Copilot solution export or Claude settings/skill file).
10. Reuse ledger: a running count of intake decisions routed to reuse/improve/merge with links — your cheapest, most persuasive value metric.

## Top 10 risks

1. Bus factor of one — repo, knowledge, and approvals all route through you. (Mitigation: 60-day org hosting + second maintainer.)
2. DevTools capture breaks on any Studio refactor — fragile foundation for drift detection. (Adapter design; adopt native versioning when it ships.)
3. Atlassian ships versioning/registry/sprawl features that make parts of the workbench look redundant — the *record* stays valuable, but the pitch must anticipate this.
4. Process weight drives builders to shadow agents — the exact failure the workbench exists to prevent. (Risk tiers; paved road faster than bypass.)
5. Manual measurement decays after novelty; value story dries up. (Platform analytics + small honest samples.)
6. Pilot produces mediocre results and governance gets blamed instead of agent quality. (Pick the pilot agent for demonstrable pain, not availability.)
7. Rovo's individual-only agent access means "restricted" agents aren't reliably restricted — never treat Studio visibility as a security control.
8. A Tier 3-adjacent use case (claims/underwriting-touching) slips through without compliance review — regulatory exposure under NAIC/CDI expectations. (Write-capability + data-classification fields; escalation trigger.)
9. Governance repo doc sprawl contradicts itself and erodes trust in the source of truth. (Canonical-doc + pointer pass; run your own consistency checks on it.)
10. Positioning drift: if the capability reads as compliance overhead rather than delivery acceleration, funding and goodwill evaporate. (Every artifact leads with the pain solved; snapshot leads with adoption and time returned.)
