# Request or Improve an Agent

Use this page when no approved ROVO agent fits your workflow, or when an approved agent needs to be improved.

This page is for everyday users, leads, AI champions, agent owners, and reviewers. It keeps the first step simple: describe the workflow problem and route it to the lightest useful solution.

## Start Here

Bring the request to your lead or AI champion when possible, then use the ROVO Design Agent or `Agent Fit Intake` to shape the need.

The first question is not "What agent should we build?" The first question is "What workflow problem are we trying to solve?"

## Request Path

| Step | Action | Output |
|---|---|---|
| 1 | Describe the workflow problem, audience, trigger, and expected output. | Clear problem statement. |
| 2 | Check `User Guide - Using Approved Agents` and `Agent Inventory`. | Existing approved agent, pilot-only agent, draft candidate, or no match. |
| 3 | Ask whether the need can be handled by a better prompt, default ROVO behavior, automation, or Confluence template. | Least-complex fit option. |
| 4 | If an agent already exists, compare reuse, improvement, or extension before proposing a new agent. | Reuse / Improve / Merge / Reject / New decision. |
| 5 | If the need is agent-shaped, run `Agent Fit Intake` with the ROVO Design Agent. | Fit decision and missing evidence. |
| 6 | If documentation or source quality is the blocker, route to Agent Doc Steward or source owners. | Source cleanup or readiness actions. |
| 7 | If a pilot or team-approved path is requested, prepare review evidence instead of assuming approval. | Promotion packet, feedback tracker, or reviewer decision. |

## Possible Outcomes

| Outcome | Use When |
|---|---|
| Use an approved agent | A source-backed approved agent already fits the workflow and audience. |
| Use a prompt | The need is narrow, individual, or one-time. |
| Use an automation | The workflow is repeatable, rule-based, scheduled, or deterministic. |
| Improve an existing agent | An agent exists but needs better instructions, sources, examples, or evaluation coverage. |
| Extend an existing agent | A governed agent is the right base but needs a reviewed team or workflow extension. |
| Use or create a Confluence template | The real gap is inconsistent source material or repeatable documentation structure. |
| Keep as an agent candidate | The idea may become an agent but needs discovery before build. |
| Create a new governed agent | The workflow is repeatable, shared, valuable, source-backed, measurable, and owned. |
| Not ready | Ownership, sources, permissions, measurement, or risk controls are unclear. |

## Improvement Requests

For an existing agent, capture:

| Field | What To Capture |
|---|---|
| Agent | Current agent name. |
| User group | Team, role, or audience affected. |
| Current problem | What the agent cannot do well enough today. |
| Source evidence | Pages, Jira issues, examples, or evaluation results that show the gap. |
| Desired behavior | What should change. |
| Risk | What could go wrong if the change is made badly. |
| Owner or reviewer | Lead, AI champion, agent owner, or reviewer who can help triage. |

## Build Or Promotion Requests

Do not treat a request as approval to build, pilot, or promote an agent.

Before a durable new agent or extension moves forward, the owner or helper should collect:

- Problem statement.
- Intended users or audience.
- Existing agents or pages checked.
- Fit decision.
- Duplicate decision.
- Owner and backup owner, or a documented ownership gap.
- Knowledge sources needed.
- Tools, actions, or skills needed.
- Context readiness status.
- Measurement idea.
- Readiness risks or open questions.

Use `Context Readiness Checklist`, `Agent Inventory`, and `Measurement and Value Tracking` before treating the work as ready for pilot or team use.

## Human-Owned Decisions

Humans still own:

- final approval or rejection of an agent
- pilot or team promotion
- owner and backup owner assignment
- Studio configuration saves, visibility, managers, tools, or knowledge sources
- Jira and Confluence writes
- release, legal, compliance, privacy, personnel, or go/no-go decisions

Agents can draft, summarize, compare, and check evidence. They do not grant approval.
