# User Guide - Using Approved Agents

Use this page when you want to find and use an approved ROVO agent in the Rovo Agent Inventory space.

This guide is for end users, including pilot users. You do not need to understand Project Brain pages, Studio configuration, or governance templates before using an approved agent. You do need to confirm that the agent is approved for your audience and that you can review its output before acting on it.

## What Counts As Approved

An agent should appear on the approved-user list only when repo-backed or approved Confluence evidence shows that it is approved for the intended audience.

| Status | Everyday user handling |
|---|---|
| Active | May appear here only when readiness and audience are source-backed for everyday use. |
| Team Approved | May appear here only for the named team or audience. |
| Pilot Approved | Pilot-only. Use only if you are in the named pilot audience. |
| Experimental, Draft, In Review, Needs Cleanup, Candidate, Archived, Superseded, or Historical Reference | Not an approved everyday-user agent. Use the request or improvement path instead. |

## How To Use An Approved Agent

1. Find the agent on this page or from a direct lead/champion recommendation.
2. Confirm the agent is approved for your team, role, or workflow.
3. Start with a clear workflow need, not a vague request.
4. Give the agent scoped source context, such as the specific Jira issue, Confluence page, file, or approved source packet.
5. Ask for draft help, review support, summarization, or gap checking within the agent's purpose.
6. Review the output before using it.
7. Record feedback when the output is useful, wrong, incomplete, unsafe, or needs improvement.

## Safe Use Rules

- Do not treat an agent response as final approval.
- Do not ask an agent to make go/no-go, legal, compliance, release, personnel, or ownership decisions.
- Do not ask an agent to update Jira, Confluence, ROVO Studio, repositories, or workflow records unless that write path has a separate approved governance decision.
- Do not paste secrets, tokens, cookies, private exports, raw HARs, or sensitive data unless the agent and source handling are explicitly approved for that data.
- If the agent does not have enough source context, ask it to mark the gap instead of guessing.

## When The Agent Is Not Useful

Use `Request or Improve an Agent` when:

- You cannot find an approved agent for your workflow.
- The approved agent gives incomplete, stale, confusing, or overconfident output.
- The agent seems to need better source pages, examples, prompts, or evaluation tests.
- You think an existing agent should support your team with a safe extension.
- You are not sure whether the answer should be an agent, prompt, automation, template, or source cleanup.

## Feedback To Capture

When reporting feedback, include:

| Field | What To Provide |
|---|---|
| Agent | Name of the agent used. |
| Workflow | What you were trying to do. |
| Source context | Jira issue, Confluence page, source packet, or other approved input used. |
| Result | Accepted, edited, rejected, or not usable. |
| Issue | Missing source, wrong answer, confusing output, unsafe suggestion, or improvement idea. |
| Impact | Time saved, rework created, quality improvement, or risk avoided. |

Keep private source material out of feedback channels unless the channel is approved for that data.
