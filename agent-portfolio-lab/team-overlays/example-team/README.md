# Example Team Overlay

This overlay shows where team-specific release conventions belong. Keep reusable core files generic; put team-specific Jira fields, naming, branches, sources, approvals, and cadence here.

## Overlay Inputs

- `jira-field-map.md`
- `release-naming.md`
- `branch-conventions.md`
- `knowledge-sources.md`
- `approval-workflow.md`
- `release-cadence.md`

## Use

When testing an agent, include this overlay after reusable core instructions. Treat values here as examples that must be replaced for a real team.
