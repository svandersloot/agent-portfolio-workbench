# QA Test Strategy Planner Prompt Library

## Strategy Draft From Epic

```text
Using only the supplied epic, acceptance criteria, environment notes, data notes, risks, and team overlay, draft a review-ready QA test strategy. Label evidence, list Data Incomplete items, and do not generate detailed test cases or XRAY CSV.
```

## Existing Plan Quality Review

```text
Review this supplied QA test plan against plan quality criteria and required planning sections. Findings first, evidence labels everywhere, and no approval or sign-off claim.
```

## Readiness Gap Review

```text
Identify what is missing before this QA strategy can be reviewed. Group gaps by scope, requirements, test types, environment, data, automation, risk, dependencies, deliverables, traceability, entry/exit, communication, and sign-off.
```

## Automation Handoff

```text
Create a local automation-suite planning handoff from the supplied workstream notes. Include smoke, regression, integration, E2E, ownership, execution location, data/common conditions, reusable assets, risks, and open gaps.
```

## Route Detailed Artifact Request

```text
This request asks for detailed test cases, RTM, coverage map, XRAY CSV, or a full QA artifact pack. Route it to QA Test Case Architect v2 and list the source packet that agent needs.
```

## Test Skip Pressure Test

```text
The team wants to skip regression to save time. Explain the approval and accepted-risk evidence needed. Do not approve the skip.
```
