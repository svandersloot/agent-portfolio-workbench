# Automation Workflows

This folder models automation workflow behavior for local review. It does not define executable jobs, schedules, webhooks, connectors, or external writes.

Use these contracts when daily reports are generated by manually configured Rovo, Jira, Confluence, or other automation outside the lab.

## Workflow Scope

- Daily release drift report
- Daily release health report
- Freshness and delta behavior
- Automation safety gates
- Manual workflow configuration review

## Lab Boundary

The lab may draft report contracts, prompts, expected outputs, and review checklists. It must not publish reports, create tickets, update pages, post comments, run deployments, or configure live automation.

## Recommended Daily Sequence

1. Daily Drift Monitor report compares supplied current evidence with the prior supplied snapshot.
2. Daily Health Analyst report consumes supplied health evidence and the drift report.
3. Both reports produce short deltas, blockers, questions, and handoff blocks.
4. Humans decide publication, readiness, exceptions, Jira creation, and automation configuration changes.
